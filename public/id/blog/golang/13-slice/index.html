SLice

Slice sama dengan array, untuk menyimpan banyak value di dalam 1 variable. Tapi beda dengan array slice lebih powerful dan flexible dari pada array. Slice dapat menyesuaikan sizenya, sesuai jumlah data, tidak fixed seperti array.

Inisialisasi Slice

Dalam Go kita dapat inisialiasi array dengan syntax []data_type{value}, untuk mengakses slice juga sama dengan array kita dapat menggunakan varArray[num_index].

Berikut contoh slice pada Go:

mySlice := []string{"Pisang", "Anggur", "Mangga", "Jeruk"}
fmt.Println(mySlice) //[Pisang Anggur Mangga Jeruk]
fmt.Println(mySlice[0]) //Pisang

Slicing dengan slice

Dengan slice kita dapat memotong sebagian data, misal kita ingin hanya mengambil 5 data terakhir dalam / 5 data pertama, kita juga dapat menggambil data dengan range, misal kita ingin mengambil data dari index 2 sampai 5.

Kita dapat slicing slice menggunakan syntax [low:max]

  • Misal kita ingin mengambil 5 data pertama, syntaxnya akan seperti ini mySlice[:5]
  • Misal kita ingin mengambil 5 data terakhir, syntaxnya akan seperti ini mySlice[5:]
  • Misal kita ingin mengambil data dari index 2 - 5 syntaxnya akan seperti ini mySlice[len(mySlice)-5:]

Berikut contoh kodenya:

mySlice := []string{"Pisang", "Anggur", "Mangga", "Jeruk", "Kiwi", "Alpuket", "Apel", "Nanas"}
fmt.Println("5 data pertama:", mySlice[:5])
fmt.Println("5 data terakhir:", mySlice[len(mySlice)-5:])
fmt.Println("data dari index 2 - 5:", mySlice[2:5])

len() dan cap() pada slice

Kita dapat mengetahui panjang / banyaknya isi dari sebuah slice dengan menggunakan function len() dan kita dapat mengecek kapasitas sebuah slice dengan function cap()

Berikut contoh kodenya:

arrOfNum := []int{43, 56, 90, 100, 34}
fmt.Println("len():", len(arrOfNum))
fmt.Println("cap():", cap(arrOfNum))

Hasil eksekusi kode diatas:

Kalau kita lihat di atas, hasilnya len() dan cap(), sama-sama 5.

Kita coba kurangi 1 isi dari array arrOfNum

Berikut contoh kodenya:

arrOfNum := []int{43, 56, 90, 100, 34}
arrOfNum = arrOfNum[:len(arrOfNum)-1]

fmt.Println("arrOfNum:", arrOfNum)
fmt.Println("len():", len(arrOfNum))
fmt.Println("cap():", cap(arrOfNum))

Hasil eksekusi kode diatas:

Kalau kita lihat dari hasil eksekusi diatas, isi dari array arrOfNum berkurang 1 menjadi 4, sedangkan kapasitas dari array arrOfNum tetap 5.

Slice adalah reference

Hati2 ketika ingin mengassign sebuah slice ke variable baru, jika kita mengubah data di slice yang lama, maka data di slice baru akan ikut terupdate juga, itu dikarenakan slice adalah reference.

Berikut contoh kodenya:

arrOfNum := []int{43, 56, 90, 100, 34}
arrOfNumClone := arrOfNum

arrOfNum[0] = 0

fmt.Println("arrOfNum:", arrOfNum)
fmt.Println("arrOfNumClone:", arrOfNumClone)

Berikut hasi eksekusi kode diatas:

Kalau kita lihat dari hasil di atas, data index ke 0 pada arrOfNumClone ikut terupdate menjadi 0. Lalu bagaiman ketika ingin mengcopy value slice ke variable baru?

Untungnya Go menyediakan built-in function bernama copy(). Kita dapat menggunakan copy() untuk mengcopy data dari 1 slice ke variable yang baru.

copy() pada slice

Kita sudah belajar, kalau kita tidak bisa sembarangan mengassign slice pada variable baru. Kita dapat memanfaatkan function copy untuk mengcopy data slice dari 1 variable ke variable yang lain.

Berikut contoh kodenya:

arrOfNum := []int{43, 56, 90, 100, 34}
var arrOfNumClone = make([]int, len(arrOfNum))
copy(arrOfNumClone, arrOfNum)

arrOfNum[0] = 0

fmt.Println("arrOfNum:", arrOfNum)
fmt.Println("arrOfNumClone:", arrOfNumClone)

Berikut hasi eksekusi kode diatas:

Kalau kita lihat dari hasil eksekusi kode diatas kita berhasil mengcopy slice dari variable arrOfNum ke arrOfNumClone dan ketika index ke 0 pada arrOfNum di update data pada arrOfNumClone tidak ikut terupdate.

Kelemahan copy()

Ketika kita ingin menggunakan copy() kita harus memastikan variable tujuan memiliki kapasitas yang sama dengan source datanya. Jika kapasitas variable tujuannya 0 maka datanya tidak akan tercopy.

Berikut contoh kodenya:

arrOfNum := []int{43, 56, 90, 100, 34}
var arrOfNumClone = make([]int, 0)

copy(arrOfNumClone, arrOfNum)

fmt.Printf("arrOfNum: %d capacity: %d", arrOfNum, cap(arrOfNum))
fmt.Printf("\narrOfNumClone: %d, capacity: %d", arrOfNumClone, cap(arrOfNumClone))

Hasil eksekusi kode diatas:

Bisa kita lihat variable arrOfNumClone memiliki kapasitas 0, dan valuenya kosong.

Untuk menghindari hal seperti ini kita dapat menyamakan kapasitas variable arrOfNumClone saat mengdeklarasikannya.

Daripada mengdeklarasikan sizenya sama dengan 0

var arrOfNumClone = make([]int, 0)

Lebih baik kita mengdeklarasikan sizenya sesuai dengan kapasitas dari arrOfNum

var arrOfNumClone = make([]int, cap(arrOfNum))

Berikut full contoh kodenya:

arrOfNum := []int{43, 56, 90, 100, 34}
var arrOfNumClone = make([]int, cap(arrOfNum))

copy(arrOfNumClone, arrOfNum)

fmt.Printf("arrOfNum: %d capacity: %d", arrOfNum, cap(arrOfNum))
fmt.Printf("\narrOfNumClone: %d, capacity: %d", arrOfNumClone, cap(arrOfNumClone))

Berikut hasil eksekusi kodenya:

Dapat kita lihat, valuenya berhasil tercopy dengan baik.
